## Introduction

### C'est quoi ?

[Ollama](https://ollama.com/) est une application qui permet de faire tourner et de manipuler une IA (des *modèles de langage*) sur son propre matériel. [Open WebUI](https://github.com/open-webui/open-webui) fournit une interface web pour interagir avec Ollama.

> Ollama est un *outil* léger et extensible pour la construction et l'exécution de modèles de langage fournis localement. Il expose une API simple pour créer, exécuter et gérer des modèles, ainsi qu'une bibliothèque de modèles préconstruits qui peuvent être facilement utilisés dans une variété d'applications.

:::note[Pourquoi Ollama et OpenWebUI ?]
Ollama s'est imposé comme standard dans la communauté. Il est très facile d'utilisation et fournit des IA "clef en main". OpenWebUI en est directement dérivé comme interface utilisable, Ollama ne fournissant qu'une console locale.
:::

### Avantages & inconvénients

**Avantages :**
- Les données ne sortent pas du serveur
- On peut utiliser n'importe quel modèle disponible, et ils sont nombreux
- C'est une bonne manière d'apprendre à utiliser l'outil "IA"
- Connecté à Home Assistant, ça fait une IA *à la Jarvis*

**Inconvénients :**
- Les LLM consomment énormément de ressources (VRAM, GPU, ...)
- ... Et d'électricité !
- Seuls les modèles ayant des licences permissives sont disponibles

### Alternatives

- [ChatGPT](https://chatgpt.com) : propriétaire, hébergé
- [Le Chat](https://chat.mistral.ai/chat) : modèle open-source, mais service hébergé
- [LocalAI](https://localai.io) : open-source, auto-hébergé

## Installation

### Prérequis

- Une machine (virtuelle ou non) sous debian avec *au moins* 4 cœurs et 10 Go de DDR
- Docker
- Une ou des carte(s) graphique(s) (du même fabricant !)

Avant d'installer quoi que ce soit, on attribue si possible une adresse IP fixe. Ensuite, on vérifie que la carte graphique est bien détectée :

```bash
lspci
```

Qui devrait renvoyer quelque chose du genre :

```
00:10.0 VGA compatible controller: NVIDIA Corporation GP102 [GeForce GTX 1080 Ti] (rev a1)
00:10.1 Audio device: NVIDIA Corporation GP102 HDMI Audio Controller (rev a1)
```

### Installation des drivers GPU

#### NVIDIA

Les drivers nvidia étant propriétaires, il faut activer les dépôts contrib et non-free *avant* d'installer `nvidia-detect` :

```bash
sudo apt install nvidia-detect
nvidia-detect
```

En fin de commande, il indique quel paquet installer avec `apt install <nomDuPaquet>`.

Par défaut, docker est incapable d'utiliser directement le GPU. Il faut installer le toolkit nvidia :

```bash
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey \
  | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
```

```bash
curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list \
  | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' \
  | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
```

```bash
sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

#### AMD

Les pilotes sont inclus dans le noyau Linux par défaut, rien à faire.

:::caution
Certains utilisateurs rapportent le besoin d'installer le package "ROCm" sur certaines configurations.
:::

### Docker Compose

La méthode recommandée est de passer via Docker. On sépare Ollama et OpenWebUI pour les exposer indépendamment (Ollama peut servir d'assistant via Home Assistant).

#### GPU NVIDIA

```yaml
services:
  ollama:
    container_name: ollama
    image: ollama/ollama:latest
    ports:
      - 11434:11434
    volumes:
      - ./ollama:/root/.ollama
    restart: unless-stopped
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities:
                - gpu

  open-webui:
    container_name: open-webui
    image: ghcr.io/open-webui/open-webui:cuda
    ports:
      - 3000:8080
    volumes:
      - ./open-webui:/app/backend/data
    environment:
      - 'OLLAMA_BASE_URL=http://ollama:11434'
    restart: always
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities:
                - gpu
```

:::caution[Drivers Docker NVIDIA]
Les drivers *seuls* ne suffisent pas ! Ne pas oublier d'installer le toolkit docker nvidia (voir ci-dessus).
:::

#### GPU AMD

```yaml
services:
  ollama:
    container_name: ollama
    image: ollama/ollama:rocm
    volumes:
      - ollama:/root/.ollama
    ports:
      - 11434:11434
    devices:
      - /dev/kfd
      - /dev/dri

  open-webui:
    container_name: open-webui
    image: ghcr.io/open-webui/open-webui:main
    ports:
      - 3000:8080
    volumes:
      - ./open-webui:/app/backend/data
    environment:
      - 'OLLAMA_BASE_URL=http://ollama:11434'
    restart: always
```

:::note[GPU Intel]
Officiellement, les cartes Intel ne sont pas prises en charge.
:::

Après le docker compose :

```bash
docker compose up -d
```

### Installation par exécutable

:::caution[Bonnes pratiques]
L'installation par le script automatique n'est recommandée que dans le cas d'une utilisation sur une machine dédiée.
:::

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Puis pour installer OpenWebUI :

```bash
sudo apt install python3-pip -y
pip install open-webui --break-system-packages
open-webui serve
```

## Configuration

### Vérifications et compte administrateur

Une fois installé, vérifiez qu'Ollama tourne sur `http://IP:11434`. Il devrait renvoyer :

```
Ollama is running
```

Rendez-vous sur `http://IP:3000`. Cliquez sur "Commencer", renseignez les champs requis. Le premier compte créé sera administrateur.

### Choisir son modèle

Connecté, cliquez sur l'icône de compte → **Panneau d'administration** → **Paramètres** → **Modèles** → **Gérer les modèles**. Vérifiez qu'Ollama est bien renseigné sur `http://ollama:11434`. Choisissez un modèle sur [ollama.com/models](https://ollama.com/models).

:::caution[Taille du LLM = VRAM]
L'ensemble du LLM doit être chargé dans la carte graphique. Plus le LLM possèdera de paramètres (3b, 8b, 24b, ...) plus il sera précis mais aussi gourmand. Je recommande des versions 8b d'environ 4 à 6 Go.
:::

:::tip[IA décensurée]
J'utilise des IA *oblitérées*, réentraînées sans alignement moral. La série `dolphin` basée sur `Deepseek` *fine-tuned* avec `Mistral` est capable de raisonnements avancés.
:::

Téléchargez le modèle, actualisez la page, et il sera reconnu. Vous pouvez alors le customiser (nom, prompt système...).

### Ajouter la recherche en temps réel

Par défaut, le LLM se limite aux données de son entraînement. On peut l'augmenter avec la recherche web en temps réel.

Dans le menu d'administration → **Recherche Web** :
1. Activez la fonctionnalité
2. Sélectionnez **DuckDuckGo** (pas besoin de clef API, respectueux de la vie privée)
3. Recherches concurrentes : `5`
4. Premiers résultats : `3`

Lors d'une conversation, cliquez sur "Recherche Web" pour activer la recherche en temps réel.
