## Introduction

Ce guide détaille la création d'un onglet météo complet dans Home Assistant, avec météo en temps réel, prévisions, qualité de l'air et cartes radar.

### Mise en page

J'utilise le mode "sections" avec 4 colonnes et l'en-tête. La troisième colonne est en fait double : les cartes météo gagnent à prendre plus d'espace.

---

## L'en-tête (Header)

L'en-tête est la première chose qu'on voit : idéale pour les alertes météo ou les allergies. On crée des **cartes conditionnelles** qui ne s'affichent que lorsque l'entité est active.

:::tip[Source]
J'utilise l'intégration de l'[IRM](https://github.com/jdejaegh/irm-kmi-ha) (Belgique). Pour les autres, [AccuWeather](https://www.home-assistant.io/integrations/accuweather/) fournit les concentrations et le niveau de danger.
:::

Chaque pollen possède son propre badge :

```yaml
type: entity
show_name: true
show_state: false
show_icon: true
entity: sensor.maison_grasses_level
name: Graminées
show_entity_picture: false
color: pink
visibility:
  - condition: state
    entity: sensor.maison_grasses_level
    state_not: none
  - condition: state
    entity: sensor.maison_grasses_level
    state_not: unavailable
  - condition: state
    entity: sensor.maison_grasses_level
    state_not: unknown
```

## Première colonne : Météo temps réel et prévisions

### Météo actuelle

La météo actuelle est donnée par l'intégration [IRM](https://github.com/jdejaegh/irm-kmi-ha) (disponible sur HACS). Les informations manquantes sont récupérées par [tomorrow.io](https://www.home-assistant.io/integrations/tomorrowio/) et [AccuWeather](https://www.home-assistant.io/integrations/accuweather/). La carte utilisée est [Platinum Weather Card](https://github.com/tommyjlong/platinum-weather-card) (HACS).

```yaml
custom1_icon: mdi:cloud
custom1_units: "%"
custom1_value: sensor.maison_couverture_nuageuse
entity_apparent_temp: sensor.maison_temperature_apparente
entity_fire_danger: sensor.tomorrow_io_maison_indice_d_incendie
entity_forecast_icon: weather.maison
entity_forecast_max: sensor.maison_realfeel_temperature_max_day_0
entity_forecast_min: sensor.maison_realfeel_temperature_min_day_0
entity_humidity: sensor.namur_belgium_humidite
entity_pop: sensor.maison_precipitation
entity_pressure: sensor.maison_current_pressure
entity_summary: sensor.maison_condition_day_0
entity_sun: sun.sun
entity_temperature: sensor.maison_temperature
entity_uv_alert_summary: sensor.maison_current_uv_index
entity_wind_bearing: sensor.maison_current_wind_bearing
entity_wind_gust: sensor.maison_vitesse_des_rafales
entity_wind_speed: sensor.maison_current_wind_speed
option_locale: fr
section_order:
  - overview
  - extended
  - slots
  - daily_forecast
show_section_extended: false
slot_l1: forecast_max
slot_l2: pressure
slot_l3: sun_next
slot_l4: wind
slot_r1: forecast_min
slot_r2: popforecast
slot_r3: sun_following
slot_r4: custom1
text_card_title: Météo actuelle
type: custom:platinum-weather-card
```

### Prévisions

Les prévisions sont séparées en deux cartes, fournies par [Weather Chart Card](https://github.com/mlamberts78/weather-chart-card) (HACS) :

- **Prochaines heures** — données de [tomorrow.io](https://www.home-assistant.io/integrations/tomorrowio/)
- **Prochains jours** — données d'[AccuWeather](https://www.home-assistant.io/integrations/accuweather/)

```yaml
type: custom:weather-chart-card
entity: weather.tomorrow_io_maison_daily
show_main: false
show_temperature: false
show_current_condition: false
show_attributes: false
forecast:
  precipitation_type: rainfall
  show_probability: false
  labels_font_size: "11"
  style: style1
  show_wind_forecast: true
  condition_icons: true
  type: hourly  # changer en "daily" pour les jours
  number_of_forecasts: "0"
```

## Deuxième colonne : Environnement extérieur

### Indice UV

Jauge avec l'indice UV max et actuel, mise à jour toutes les heures via [Open-UV](https://www.home-assistant.io/integrations/openuv/). Carte : [Gauge Card Pro](https://github.com/benjamin-dcs/gauge-card-pro).

```yaml
type: custom:gauge-card-pro
segments:
  - from: 0
    color: var(--green-color)
  - from: 3
    color: rgb(255, 255, 0)
  - from: 5
    color: "#FFA500"
  - from: 7
    color: red
needle: true
gradient: true
min: 0
max: 10
titles:
  primary: Indice UV
entity: sensor.openuv_indice_uv_actuel
entity2: sensor.openuv_indice_uv_maximal
```

:::caution[Pas d'auto-update]
L'entité n'est pas mise à jour automatiquement. Créez une automatisation :

```yaml
alias: Update OpenUV
triggers:
  - trigger: time_pattern
    minutes: /60
conditions:
  - condition: numeric_state
    entity_id: sun.sun
    value_template: "{{ state.attributes.elevation }}"
    above: 10
actions:
  - action: homeassistant.update_entity
    target:
      entity_id: sensor.openuv_indice_uv_actuel
```
:::

### Qualité de l'air

Données de [WAQI](https://www.home-assistant.io/integrations/waqi/). Mise en forme : pile verticale avec [Bubble Card](https://github.com/Clooos/Bubble-Card) pour les concentrations et [Apex Charts Card](https://github.com/RomRider/apexcharts-card) pour le graphique 24H.

```yaml
type: vertical-stack
cards:
  - type: heading
    icon: mdi:radioactive
    heading: Pollution de l'air
    heading_style: subtitle
    badges:
      - type: entity
        show_state: true
        show_icon: false
        entity: sensor.namur_belgium_dominant_pollutant
        name: Polluant principal
        color: accent
  - type: horizontal-stack
    cards:
      - type: custom:bubble-card
        card_type: button
        button_type: state
        entity: sensor.namur_belgium_pm2_5
        name: PM2.5
      - type: custom:bubble-card
        card_type: button
        button_type: state
        entity: sensor.namur_belgium_pm10
        name: PM10
      - type: custom:bubble-card
        card_type: button
        button_type: state
        entity: sensor.namur_belgium_ozone
        name: Ozone
      - type: custom:bubble-card
        card_type: button
        button_type: state
        entity: sensor.namur_belgium_nitrogen_dioxide
        name: Nitrés
  - type: custom:apexcharts-card
    header:
      show: false
    series:
      - entity: sensor.namur_belgium_pm2_5
        name: PM2.5
      - entity: sensor.namur_belgium_pm10
        name: PM10
      - entity: sensor.namur_belgium_ozone
        name: O3
      - entity: sensor.namur_belgium_nitrogen_dioxide
        name: NO2
```

## Troisième et quatrième colonnes : Cartes en temps réel

J'utilise les cartes en temps réel de [Windy](https://embed.windy.com/config/map) : vents, pluie, températures et risques incendies, intégrées par iframe.

```yaml
type: iframe
url: >-
  https://embed.windy.com/embed.html?type=map&location=coordinates&metricRain=mm&metricTemp=°C&metricWind=m/s&zoom=9&overlay=wind&product=ecmwf&level=surface&lat=50.45&lon=4.9&detailLat=50.45&detailLon=4.9&marker=true&message=true
grid_options:
  columns: 12
  rows: 6
```

:::note
Changez `overlay=wind` par `overlay=rain`, `overlay=temp` ou `overlay=fwi` pour les autres cartes.
:::
